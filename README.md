# CrimeNet — Intelligent CDR Forensics Platform

> An open, explainable forensic analysis platform for criminal network detection, behavioral profiling, and investigative intelligence from Call Detail Records (CDRs).

---

## Overview

Call Detail Records (CDRs) are metadata logs generated by telecom operators for every call, SMS, and data session. They contain no call content — only forensic metadata: who called whom, when, from where, using which device. This platform transforms raw CDR data into actionable investigative intelligence.

CrimeNet addresses three core forensic requirements:

| Requirement | Implementation |
|---|---|
| Call Pattern Analysis | Temporal profiling, frequency analysis, anomaly detection |
| Link Analysis | Social Network Analysis with centrality metrics and community detection |
| Communication Mapping | Cell tower geolocation, movement trajectory, co-location detection |

---

## Research Contributions

- **Burner Phone Detection** — IMEI-SIM correlation to detect device switching evasion tactics
- **Co-location Analysis** — Identify physical meetings between suspects with no direct call history
- **Network Dismantling Simulation** — Graph-theoretic approach to prioritize arrests for maximum network disruption
- **Explainable Forensics** — Every flag is traceable to raw CDR rows (court-admissible reasoning)
- **Criminal Cell Detection** — Louvain community detection to identify organized sub-groups

## Quick Start

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Generate a demo dataset

```bash
# Gang scenario — 500 records, Karnataka towers
python utils/cdr_generator.py --records 500 --scenario gang --state KA --output data/sample/gang_demo.csv

# Fraud scenario — 300 records
python utils/cdr_generator.py --records 300 --scenario fraud --state MH --output data/sample/fraud_demo.csv

# Burner phone scenario — reproducible
python utils/cdr_generator.py --scenario burner --seed 42 --output data/sample/burner_demo.csv
```

### 3. Launch the dashboard

```bash
streamlit run dashboard/app.py
```

Open `http://localhost:8501` in your browser.

## Analysis Modules

### `analysis/ingest.py`
- Load and validate CDR CSV files
- Normalize phone numbers, timestamps, tower IDs
- Detect and report data quality issues

### `analysis/network.py`
- Build directed weighted communication graph (NetworkX)
- Compute: Degree, Betweenness, PageRank centrality
- Community detection: Louvain algorithm
- Network dismantling: centrality-guided node removal simulation

### `analysis/geo.py`
- Map Tower IDs to GPS coordinates
- Reconstruct suspect movement trajectories
- Co-location detection: two numbers at same tower within configurable time window
- Generate interactive Folium maps

### `analysis/temporal.py`
- Call volume by hour/day/week
- Detect night-time coordination patterns
- Communication gap analysis (silence before/after crime events)
- Burst detection: sudden spike in call volume

### `analysis/anomaly.py`
- Isolation Forest anomaly scoring on per-number call features
- Burner phone detection: IMEI-SIM mismatch tracking
- One-way communication detection
- Short-duration high-frequency call pattern (coded signalling)

## Demo Scenario

The included demo dataset simulates a **SIM card fraud ring** operating across Karnataka:

- **3 ringleaders** — high betweenness centrality, coordinate operations
- **8 intermediaries** — relay instructions, buffer the leaders
- **4 victims** — receive only, never initiate
- **2 burner phones** — activated after week 2, switch IMEI by week 4
- **Physical meetings** — co-location at Bengaluru Koramangala tower before each fraud wave

Walk through the demo narrative:
1. Load `data/sample/gang_demo.csv`
2. Network tab reveals 3 hub nodes with high betweenness
3. Geographic map shows a recurring meeting point
4. Anomaly tab flags burner phones by IMEI-SIM mismatch
5. Dismantling simulation: arrest these 3 to break 80% of network paths

## CDR Field Reference

| Field | Description | Example |
|---|---|---|
| `Calling_Number` | Originating phone number | 9876543210 |
| `Called_Number` | Destination phone number | 8765432109 |
| `Date` | Date of communication | 2024-03-15 |
| `Time` | Time of communication | 14:32:07 |
| `Duration_sec` | Call duration (0 for SMS) | 245 |
| `Call_Type` | Voice / SMS / Data | Voice |
| `Caller_IMEI` | Device identifier (hardware) | 356789012345678 |
| `Caller_IMSI` | SIM card identifier | 404100123456789 |
| `Caller_Operator` | Telecom provider | Jio / Airtel / Vi / BSNL |
| `Tower_ID` | Cell tower ID | KA-BLR-01 |
| `Tower_Location` | Tower location name | Bengaluru Koramangala |
| `Tower_Latitude` | GPS latitude | 12.9352 |
| `Tower_Longitude` | GPS longitude | 77.6245 |
